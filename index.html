EXERCISE 3: CSS Transitions

The transition property allows us to change values of elements over a period of time. This property requires a trigger to run. The trigger could be a change in the element state (example :hover), or a Javascript event. Key differences between Transitions and Animations are:

  1. Transitions require a trigger to start and Animations do not require one.
  2. Transitions can change only between two states while Animations can change between multiple states.
  3. Transitions cannot run in a loop while Animations can run as many times as we want.

EXERCISE 5: Animation Libraries

There are also a ton of animation libraries that can be used on your website to create quick animation effects. An example of such libraries is the AOS(Animate On Scroll) library.

One key benefit of using animation libraries is that they are fairly easy to set up on your website and save a lot of time coding animations in css. The disadvantage to using libraries is that we have do not have full control over the animation and they could have negative impact on website performance as the libraries have to be loaded first before they can be used.
